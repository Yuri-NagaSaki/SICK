NETS · 猫脚本

Network Endpoint Throughput Sampler

Public iperf3 tests (IPv4 / IPv6) against a curated set of providers — DataPacket, Fiberby, Kamel, BuyVM, Leaseweb, Clouvider, Eranium, and more. One curl.

iperf3 send + recv IPv4 · IPv6 22 endpoints MIT
Quick start ba.sh · catbash.net
$ curl -sL https://ba.sh/nets | bash
$ curl -sL https://catbash.net/nets | bash
$ curl -sL https://catbash.net/nets | bash -s -- -r

Needs iperf3 + python3. Pass options after bash -s --.

Examples

Common recipes
$ curl -sL https://ba.sh/nets | bash -s -- -r -t 5

Reduced set, 5s per direction — good first look.

$ curl -sL https://ba.sh/nets | bash -s -- --region apac -4

Asia-Pacific endpoints, IPv4 only.

$ curl -sL https://ba.sh/nets | bash -s -- --region eu --recv-only -t 8

Europe download-only (saves half the traffic).

$ curl -sL https://ba.sh/nets | bash -s -- -l

Print the catalog without running tests.

$ curl -sLO https://catbash.net/nets/nets.sh && chmod +x nets.sh && ./nets.sh -r

Download once, run locally (uses local endpoints.json if present).

How to read the output

SendUpload: your machine → remote iperf3
RecvDownload: remote → your machine (-R)
LinkAdvertised peer capacity (not a guarantee)
TrafficBytes transferred this run (summary)
  • M / G = Mbps / Gbps. Compare relative ranks more than absolute peaks.
  • busy = peer is running someone else's test. Re-run later.
  • fail = connect/timeout/error. Firewall, routing, or dead public node.
  • Ports are retried internally and not printed (keeps the table clean).
  • A full global dual-stack run moves a lot of data (tens of GiB). Prefer -r or --region first.

Tips

  • Be polite on public nodes: default -P 4, avoid long -t storms.
  • Results are comparative, not lab-grade capacity of your NIC or the peer DC.
  • Override catalog: NETS_ENDPOINTS_JSON=/path/to.json ./nets.sh
