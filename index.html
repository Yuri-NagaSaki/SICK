SICKv3.0.0

Server Info & Check Kit

One-shot Linux server hardware inventory. Run once, get a full colorized report — or JSON. Zero footprint.

Features

  • One-shot, zero footprint: Pure collection and rendering — nothing is written to disk, no service is installed. Run it, read it, done.
  • Aligned, colorized tables: Memory, filesystems, disks/SMART, NVMe health, RAID, network and GPU all render as boxed tables with status colors — correct even with CJK text.
  • Bilingual output: Full English and Chinese via the -cn flag.
  • JSON output: --json emits a single machine-readable object to stdout — pipe it into jq or your CMDB.
  • Privacy-aware: IPv4/IPv6 and MAC addresses are masked by default (network prefix / vendor OUI kept). Serial numbers are shown in full for inventory.
  • Friendly dependencies: Missing tools are detected and listed; the script asks Y/N before installing. Use -y to auto-install or --no-install to report with what's present.
  • Broad compatibility: Debian/Ubuntu, RHEL/CentOS/Alma/Rocky/CloudLinux, Fedora, Arch/Manjaro, openSUSE/SLES.

What it reports

  • System — Hostname, OS, kernel, uptime
  • CPU / Platform — Model, cores/threads, frequency, cache, live usage, temperature, sockets/NUMA, virtualization, microcode
  • Memory — Total / used / available, plus per-module table (size, type, speed, vendor, serial, part number)
  • Disks — Mounted-filesystem usage + SMART summary (health, power-on hours, temperature, lifetime I/O, defects)
  • NVMe deep health — Per-drive temperature, endurance used, spare, and health status
  • RAID — Software RAID (mdraid), hardware RAID/HBA controllers, vendor CLI tools
  • Network — Physical interfaces: link/speed/duplex, masked MAC & IPv4, NIC model, error/drop counters
  • GPU — Vendor and class (Discrete / Integrated / BMC); NVIDIA: memory, driver, temperature
  • Motherboard — Vendor, model, BIOS vendor/version

Usage

One-line run (default · English)

curl -sL https://ba.sh/sick | bash
wget -qO- https://ba.sh/sick | bash

Pass options via bash -s -- <options>. Prefer sudo for full data (dmidecode, SMART, sensors).

Chinese output

curl -sL https://ba.sh/sick | bash -s -- -cn

Auto-install missing tools

curl -sL https://ba.sh/sick | bash -s -- -y

JSON to stdout

curl -sL https://ba.sh/sick | bash -s -- --json

Run from source

curl -sLO https://raw.githubusercontent.com/Yuri-NagaSaki/SICK/main/hardware_info.sh
chmod +x hardware_info.sh
sudo ./hardware_info.sh

Options

Option Description
-cn, --chinese Output in Chinese
-us, --english Output in English (default)
-j, --json Print a JSON object to stdout only
-y, --yes Install missing tools without prompting
--no-install Never install; report with whatever tools are present
-h, --help Show help
-v, --version Show version

Supported systems

Debian / Ubuntu / Mint apt RHEL / CentOS / Alma / Rocky yum · dnf Fedora dnf Arch / Manjaro pacman openSUSE / SLES zypper CloudLinux dnf

Dependencies detected at startup: dmidecode, lshw, smartmontools, ethtool, nvme-cli, jq, pciutils, lm-sensors. Missing tools map to the right package for your distro and install on confirmation.

Privacy

The report masks IPv4/IPv6 (keeps the network prefix, hides host bits) and MAC addresses (keeps the vendor OUI, hides the device portion). Serial numbers (disk, memory, motherboard) are shown in full so you can inventory hardware — keep that in mind before sharing a report publicly.
