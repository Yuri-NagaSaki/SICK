Server Info & Check Kit

SICK

One command. Every piece of hardware. Zero footprint.

A one-shot Linux server hardware inventory tool. It collects everything about a box in a single run and prints a clean, colorized report — or JSON. No daemon. No persistence. No monitoring.

⬤ one-shot · nothing written to disk ⬤ bilingual EN / 中文 ⬤ JSON output ⬤ MIT
$ curl -sL https://ba.sh/sick | bash
Add -cn for Chinese · -y to auto-install tools · --json for machine output. Pass options via bash -s -- <options>.
root@server:~ — ./hardware_info.sh 
root@server:~$ curl -sL https://ba.sh/sick | bash

01

Features

⏻

One-shot, zero footprint

Pure collection and rendering — nothing is written to disk and no service is installed. Run it, read it, done.

▤

Aligned, colorized tables

Memory, filesystems, disks/SMART, NVMe health, RAID arrays, network and GPU all render as boxed tables with status colors — correct even with CJK text.

文

Bilingual output

Complete English and Chinese output via the -cn flag.

{}

JSON output

--json emits a single machine-readable object to stdout — pipe it straight into jq or your CMDB.

◈

Privacy-aware

IPv4/IPv6 and MAC addresses are masked by default (network prefix / vendor OUI kept). Serial numbers are shown in full for inventory.

⚙

Friendly dependencies

Missing tools are detected and listed; the script asks Y/N before installing anything. Use -y to auto-install or --no-install to report with what's present.

02

Broad compatibility

Debian / Ubuntu / Mint apt RHEL / CentOS / Alma / Rocky yum · dnf Fedora dnf Arch / Manjaro pacman openSUSE / SLES zypper CloudLinux dnf

Options

-cn, --chineseOutput in Chinese
-us, --englishOutput in English (default)
-j, --jsonPrint a JSON object to stdout only
-y, --yesInstall missing tools without prompting
--no-installNever install; report with whatever tools are present
-h, --helpShow help
-v, --versionShow version
